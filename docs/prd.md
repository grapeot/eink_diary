# 墨记 eink_diary — 产品需求文档（PRD）

## 一句话

一块挂在墙上的 13.3" 彩色电子纸，每两小时把"此刻你正经历的一个瞬间"画成一幅画；一天八幅连起来，就是一本不用动手写、主角是鸭哥的视觉日记。

## 是什么

屏幕显示的不是"今天做了什么"的清单，而是从你的真实数据里**挑出一个有温度的瞬间**，画成一幅**温暖的彩铅手绘绘本风**（soft warm colored-pencil storybook illustration）的画。主角是"鸭哥"——一只拟人的鸭子。八幅画跨越一天，构成一条情绪弧线，而非活动汇报。

电子纸的稀缺价值在于它的物理属性：不发光、被动常驻视野、纸感、慢刷新。这些决定了它适合"低频更新、值得反复瞥见"的内容。两小时一画的节奏用对了慢刷新（全刷十几秒无所谓），彩色用对了画作。它是用户"SecondMind 探出物理世界的一只眼"——不接收指令，把你沉淀的生活重新渲染回你眼前。

## 产品哲学：反生产力 + 未来资产

这块屏刻意**不**追求"有用、提升生产力"。它存在的理由有两层：

**反生产力的会心一笑。** 不是所有屏都该塞满有用信息。在一个高强度、多线程的工作环境里，适当引入一点"无用"的、温暖的东西，让人抬头时会心一笑——感到自己埋头做的这些事**被看见、被 appreciate** 了。这种情绪价值，恰恰是密集仪表盘给不了的。

**给未来的资产。** 一幅画的价值会随时间增值。一个月后、一年后、甚至"X 年前的今天"回看时，它要能把人**瞬间拉回当时的感觉**。这要求每幅画**specific、富含细节**——不是泛泛一只鸭子在工作，而是带着那天独有的具体物件、姿态、情绪，多到足以唤起记忆。

这两层直接约束设计：画面必须具体而非概括（否则唤不起记忆）；基调温暖、带幽默（否则换不来会心一笑）。这也是"瞬间"镜头 + 富细节 + 彩铅温暖质感的根本动机。

## 目标用户

单一用户（项目作者，鸭哥）。一个高度多线程的 builder：AI 基础设施、调研写作、课程教学、带娃、精密爱好并行。已有一块 7 寸电子纸显示 token usage、每小时刷新、经常主动看，证明"凝视自己生活的读数"是真实高频的习惯。

## 核心需求

### 主功能：定时视觉日记

- 白天 8:00–22:00，每两小时生成一幅画（一天 8 幅），当天图按日期归档成视觉编年史。
- 每幅画 = 从那个时间窗的真实数据里挑一个瞬间 → 写成画面描述 → 出图 → 刷到 E6。
- 全程无人值守（cron 驱动）。

### 辅助功能：本地图直显

- 用户可以把一张已经准备好的本地图片直接刷到 E6：`eink-diary display IMAGE`。
- 这条路径不采集数据、不调用 LLM、不生成图片，只负责把现成图片上传到 Pi display server。
- display server 统一做设备适配：收图后处理成 1200×1600、Spectra-6 七色，再全刷到屏上。
- 目标地址默认读 `.env` / 环境变量里的 `EINK_SERVER_URL`，也允许临时用 `--server-url` 覆盖。

### 三个镜头

每幅画选一个镜头看这一天，**以"瞬间"为主**：

- **瞬间（主力）**：放大一个最有张力/最具体的细节（一句话、一个想通的时刻、一封信、和孩子的片段）。最像日记、最不可替代。
- **质地（点缀）**：抽象表达一天的"感觉/形状"（碎片 vs 专注）。需全天数据才准。
- **慢变量（点缀）**：一个可追踪的内在读数（人机耦合、创造 vs 维护），带刻度、跨天可比。分量轻，适合嵌角落或偶尔成幅。

### 主角与风格

- **主角 = 鸭哥**（一只拟人的鸭子），跨幅保持一致以形成连续性与辨识度。
- **风格 = 温暖的彩铅手绘绘本风（soft warm colored-pencil storybook illustration）**，细腻彩铅笔触、柔和暖色、温馨亲密，单一场景、富含能唤起记忆的细节。不要黏土/定格/3D 质感。

### 输入数据源

读"你这个时间窗真实经历了什么"。配置驱动：在 `.env` 配置了某源的关键变量才启用，否则跳过。

| 数据源 | 取什么 | 状态 |
|--------|--------|------|
| 微信 | 我在窗口内发出的话（主时序信号）+ 同一会话局部前后文（前后各两条，去重） | 已实现 |
| AI sessions | 我和 AI 的讨论（user turns，带逐条时间戳，按窗口精确过滤） | 已实现 |
| 邮件 | 窗口内收到的邮件 | 已实现 |
| 健康量化（house/health quantification） | 睡眠/心率/活动等身体状态（可判断"没发消息时是专注还是休息"） | 候选第四源，未接 |

## 成功标准

1. 给定时间窗，系统自动拉取该窗口真实数据、挑一个瞬间、写画面描述、产出竖版 3:4 适配 1200×1600 的图、刷到 E6，全程无人值守。
2. 8:00–22:00 每两小时一幅，一天 8 幅按日期归档。
3. 数据源不可用时优雅降级（更宽窗口 / 退到上一幅 / "信号不足"占位画），不白屏不报错。
4. 给定本地图片文件，`eink-diary display IMAGE` 能通过同一 display server 路径把图刷到 E6，并在图片缺失或未配置目标时给出明确错误。
5. 作为公开 skill：仓库只含 fake 示例即可发布，私有联系人/路由/凭证不进公开仓库。

## 非目标（明确不做）

- **不做局部刷新。** E6（Spectra 6 / ACeP）介质级不支持实用局刷，自标定波形有损屏风险且收益为零（两小时一画下全刷耗时无所谓）。详见附录。
- 不做密集实时仪表盘（看了不改变行动的监控指标）。
- 不做活动清单的具象化（"今天做了 A/B/C"换皮肤）——这是被否决的范式，详见附录。
- 不做照片相框 / 高动态天文片展示（E6 六色还原不了）。
- 不做通用天气日历；天气/日历至多作为画面输入的一个信号。
- 不做多用户、云端服务、公开数据发布。
- 不做分钟级以下刷新、动画、实时交互。
- 数据源排除：Circle、Stripe、Growth Analytics、iMessage、Typefully/Twitter（业务/对外数据，与"我这一天经历了什么"无关）。

## 物理约束（必须钉住）

13.3" Waveshare E6（Spectra 6）：

- **屏幕竖放**，3:4，原生 1200×1600。所有画作 3:4 竖版。生成用 `--size 2K`（1536×2048，略大于屏幕，保留细节），**不用 4K**（生成慢且交付远大于 E6 物理分辨率）。
- 约 150 PPI，细字会被抖动噪点吃掉，最小字号约 16–18px。
- 六色（黑白红黄蓝绿）靠 dithering 模拟中间色，走色块和排版，不走照片还原。
- 全刷约 15–20 秒，无局部刷新。刷新频率时段级（两小时），绝不分钟级。

## 附录：Alternatives Considered & Why

记录走过的弯路和被否决的方案，供后人理解为何是现在这个设计。

### A1. 内容形态的两次否决

- **数据仪表盘**（把流量/收入/token 搬上墙）：被否。这类监控指标看了不改变行动，容易变成没人看的屏。
- **活动清单具象化**（"今天做了 A/B/C" → 摩天楼/地图/邮票/棋盘/博物图鉴等换皮肤）：被否。这些方向底层是同一个活动清单的可视化，做的是"汇报"不是"日记"，信息用户本来就有。（这轮实验图为本地探索产物，未入库。）

最终经"三镜头实验"（本地实验）确立：以"瞬间"为主，质地/慢变量点缀。

### A2. 主角形象

考虑过"泛泛的人"，否决——缺乏跨幅连续性。改用鸭哥（鸭子），本地实验验证可爱且有辨识度。

### A3. 不做局部刷新（完整理由）

E6（Spectra 6 / ACeP 多色电泳）介质级不支持实用局刷：彩色像素需长多阶段 waveform 做色粒分离，无法截短成黑白屏那种瞬时局刷。厂商未提供局刷 LUT，自标定波形需面板厂级设备且有永久残影/损屏风险。E-Ink 新推的 Ripple/T2000 架构能做更平滑的"局部更新"，但需专用控制器和新面板，标准 Waveshare HAT+ 不具备，且实测全刷仍约 12 秒。结论：局刷对本项目是死路，而两小时一画的形态恰好让全刷耗时变成非问题。
