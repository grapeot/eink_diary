# 墨记 eink_diary — 产品需求文档（PRD）

## 一句话

一块挂在墙上的彩色电子纸，每两小时把"你最近在干什么"画成一幅画；一天下来，这些画连起来就是一本不用动手写的视觉日记。

## 背景与动机

电子纸的稀缺价值不在于"能显示什么"——树莓派能驱动任何内容——而在于它独有的物理属性：不发光、被动常驻在视野里、纸感、刷新慢。这些属性决定了它最适合承载"低频更新、值得被反复瞥见"的内容，而不是密集实时仪表盘。

设计经历过两次收敛。最初的想法是"数据仪表盘"（把流量、收入、token 等数据搬上墙），但这类监控指标看了不改变行动，容易变成没人看的屏。第二轮收敛到"关于你自己这台人机系统的活体读数"——这个定位来自一个观察：用户已有的一块 7 寸电子纸显示 AI token usage，每小时刷新、经常主动看，它真正满足的不是控成本，而是"把自己的生活变成可观测的数据集并能凝视它"。

最终形态是把"读数"进一步演化为**视觉日记**：不是把数据画成图表，而是让 AI 理解你这个时间窗在干什么，再生成一幅意象画。这同时用对了电子纸的两个特性——慢刷新（两小时一次，全刷十几秒无所谓）和彩色（画作需要颜色）。它也最贴合用户"SecondMind 探出物理世界的一只眼"的世界观：屏不接收指令，它把你沉淀的活动重新渲染回你眼前。

## 目标用户

单一用户（项目作者）。一个高度多线程的 builder：AI 基础设施开发、调研写作、课程教学、带娃、精密爱好长尾并行。已有一块 7 寸 Seeed reTerminal E1002 显示 token usage 的成功先例，证明"凝视自己系统读数"这一习惯是真实且高频的。

## 核心需求

### 主功能：定时视觉日记

- 白天 8:00–22:00，**每两小时生成一幅画**（一天 8 幅）。
- 每幅画的输入是那个时间窗里"用户真实在干什么"，来源见下。
- 由 AI agent 把输入压缩成一段画面描述，再交给图像生成模型出图。
- 图刷到 13.3" E6 彩色电子纸上。
- 当天每幅画归档到日期目录，攒成视觉编年史。

### 输入数据源（"读你最近在干什么"）

| 数据源 | 工具 / 路径 | 状态 |
|--------|------------|------|
| 最近邮件 / 待办提醒 | `resend_email_skill`（`received list/get`） | 已验证可用 |
| 最近 AI sessions | `contexts/ai_sessions/export_sessions.py --since-date` | 已验证（opencode + claude-code + second-mind） |
| 微信聊天记录（"我说了什么" + 上下文） | `wechat_messages` skill | 待接入（明确需求） |

数据源是模块化的：每个 source 是一个可独立开关的 adapter。新增数据源不应改动渲染或调度逻辑。

采集阶段先独立成一个 **collector CLI**：跑一下 `eink-diary collect`，它把最近一个时间窗（默认前两小时）内的三类输入——收到的邮件、我在微信发出的话、我和 AI 讨论的内容——过滤、合并成一个纯文本文件，作为后续"理解→生成"的输入。collector 只负责采集与合并，不做理解、不碰图像。每个源独立降级：任一源不可用时该段标注无数据，其余照常。

### 风格基调

视觉语言（画作的统一风格）由用户从候选版本中选定。首轮已生成 5 个不同立意的样本（光河抽象 / 中文 ambient 海报 / 赛博 second mind / 禅意水墨 / 复古博物图谱），存于 workspace 的 `tmp/ambient_eink_art/`，供选型参考。

## 成功标准

1. 给定一个时间窗，系统能自动拉取该窗口的近况、生成一段画面描述、产出一幅竖版 3:4 适配 1200×1600 的图，并刷到 E6 上，全程无人值守。
2. 8:00–22:00 每两小时触发一次，一天产出 8 幅，当天图按日期归档。
3. 数据源不可用时优雅降级（用更宽的时间窗、或退到上一幅、或生成一幅"今日无足够信号"的占位画），不能白屏或刷出报错。
4. 作为公开 skill：仓库只含 fake 示例即可发布，私有联系人 / 路由 / 凭证不进公开仓库。

## 非目标（明确不做）

- **不做局部刷新。** 13.3" E6（Spectra 6 / ACeP 多色电泳）介质级不支持实用的局部刷新：彩色像素需要长多阶段 waveform 做色粒分离，无法截短成黑白屏那种瞬时局刷。厂商未提供局刷 LUT，自标定波形需面板厂级设备且有永久残影/损屏风险。E-Ink 新推的 Ripple/T2000 架构能做更平滑的"局部更新"，但需专用控制器和新面板，标准 Waveshare HAT+ 不具备，且实测全刷仍约 12 秒。结论：局刷对本项目是死路，而两小时一画的形态恰好让全刷耗时变成非问题。本项目不在这上面投入。
- 不做密集实时仪表盘（看了不改变行动的监控指标）。
- 不做照片相框 / 高动态天文片展示（E6 六色色域还原不了精修高动态图）。
- 不做通用天气日历（网上随处可见，未用到独特资产）；天气/日历至多作为画面输入的一个信号，不是主体。
- 不做多用户、不做云端服务、不做公开数据发布。
- 不做需要分钟级以下刷新、动画、实时交互的任何功能。

## 物理约束（设计时必须钉住）

13.3" Waveshare E6（Spectra 6）：

- 分辨率约 1200×1600，约 150 PPI——比手机粗，细字会被抖动噪点吃掉，舒适可读最小字号约 16–18px。
- 六色（黑白红黄蓝绿）是固定墨水点，靠 dithering 模拟中间色，不是连续色域。纯色块和排版好看，照片有明显网点。一切设计走色块和排版，不走照片还原。
- 全刷一次约 15–20 秒，带黑白翻转闪烁；无局部刷新。
- 刷新频率必须是时段级（两小时），绝不分钟级。
