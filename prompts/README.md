# Prompt 指南

视觉日记每幅画怎么写 prompt。这里是**理念和例子（enablement），不是死模板**——AI 拿到当前窗口的素材后，自己写一个新的、贴合那个具体瞬间的 prompt，而不是往固定模板里填空。

## 核心范式：瞬间，不是活动清单

经过三轮实验（见 `../experiments/`）定下的方向：

**屏幕画的是一个"瞬间"，不是"今天做了什么"的清单。** 早期走过弯路——把当天活动列成 bullet，再换皮肤（海报/地图/邮票/棋盘/博物图鉴）具象化。问题是它们底层都是同一个活动清单的可视化，做的是"汇报"不是"日记"，而且信息用户本来就有。

正确做法：synthesize 从窗口素材里**挑一个最有张力/最具体/最有情绪的瞬间**（一句让你皱眉的对话、一个想通的时刻、一封信、和孩子的一个片段），丢掉其余，把它写成一个**有主角、有场景、有情绪的画面**。日记记的是有温度的瞬间，不是勾选框。

镜头有三种（详见 `../docs/rfc.md` 的 D0）：**瞬间（主力）** / 质地（抽象氛围，点缀）/ 慢变量（仪表读数，跨天暗线，点缀）。每幅画选一个镜头，默认用瞬间。

## 主角：鸭哥

画面里的"我"是**鸭哥**——可以是一个中国中年男子，或一只拟人的鸭子（用户自称鸭哥）。**鸭子是推荐主角**：反复出现同一只鸭哥能形成跨幅辨识度和连续性，比泛泛的人更有日记的亲密感，彩色 E6 上也更可爱。同一批画里保持主角设定一致。

## 风格

柔和叙事插画 / 彩铅 storybook 风（参考 `../experiments/duck_moments_20260606/` 和 `three_lenses_20260606/moment_*`）。温暖、亲密、单一场景。

## prompt 写法（enablement，要多样化）

不要套同一个句式。每个瞬间是不同的事，prompt 也该不同——构图、视角、情绪、光线、近景远景都可以变。一批画之间要**刻意求多样**，避免八张看起来像一个模子。一个好的瞬间 prompt 通常包含：

- 主角设定（鸭哥，鸭子或中年男子，保持一批内一致）
- 这一个具体瞬间是什么（一句场景，不是活动列表）
- 情绪基调（顿悟的松弛 / 纠结 / 会心一笑 / 专注）
- "one scene only, not a summary"（强制单场景，防止滑回清单）
- 风格 + 6 色 e-ink palette + 无文字标注

## 输出规格

- 竖版 **3:4**（屏幕竖放，原生 1200×1600）。
- 尺寸 **`--size 1K`**（1K+3:4 = 960×1280，够用）。**不要用 4K**——屏幕只有 1200×1600。
- 质量 **`--quality medium`**，不要 high（成本高，E6 抖动下看不出差别）。
- 默认模型 **`gpt-image-2`**（中文渲染可靠）。Gemini 仅备选。
- E6 六色（黑白红黄蓝绿），靠色块和排版，不追求照片级还原。

## 旧模板

`01_*.md` ~ `10_*.md` 是早期"活动清单具象化"范式的方向（海报/地图/邮票/棋盘等），已**降级为备选**。它们不是主力，保留备查。
